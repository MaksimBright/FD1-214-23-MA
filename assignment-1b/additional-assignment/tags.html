Short description of some tags

<dl> <dt> <dd> tags

There are 3 first tags that are united in one group to name something and give it a definition.

<dl>
This element stands for opening a group to insert a names and definitions inside
<dt>
This one stands for declaring a term
<dd>
Last one stands for a definition of a term itself

But these tags are not the only ones to work with textual information in html.

<pre> tag

But these tags are not the only ones to work with textual information in html. There is another one called <pre> tag. And this one allows to preserve a text inside of it in a format that is actually written by a developer.

<code> tag

This element is used to show code blocks on the page with a special font, but spaces and line breaks are not preserved.

Example of <code> tag

Same text but with <p> tag (linebreaks preserved with <br> tag)

margin: 5px 0 20px 0;

overflow: auto;
}

The <tt> tag uses "monospace" font for text declared inside. But it is already deprecated, and it will be a bad practice to use it. Instead, you can use <kbd> tag, which also applies "monospace" font.

Both tags used to qute some statement, but behave a bit differently. <blockquote> tag uses indentation at the beginning of each line and <q> tag just wraps the text into double quotes.

These tags are used to mark some parts of a text as inserted new, or deleted. It also contains useful attributes such as "cite" is a link for description of what and why was inserted or deleted and "datetime" to preserve date and time of specified format when formatting was done.

<nobr> and <wbr> tags

These tags are also used for text formatting, they are responsible for word breaks and no breaks in provided text. If really long text is wrapped with <nobr> tag it will cause a scroll bar appearance at the bottom of the page. Otherwise, if you use <wbr> to mark places where text should be started with a new line break if needed.
